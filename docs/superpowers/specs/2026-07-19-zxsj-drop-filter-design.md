# 诛仙世界秘境掉落筛选设计

## 目标

以 Tauri 2 + React + TypeScript + shadcn/ui 重做现有 Windows 掉落筛选工具，完整覆盖筛选、推荐宝鉴、概率计算、命中明细、掉落表编辑、导入导出和复制摘要，并提供可公开维护的开源项目结构。

## 已确认的产品设计

- 主窗口采用三栏工作台：左侧筛选、中央结果表、右侧明细。
- 默认视觉为“青玉纸本”：浅色中性底、青玉强调色、灰蓝边界、少量状态色。
- 左侧支持属性、属性匹配模式、部位和限定副本筛选；筛选项可为空。
- 中央提供“推荐宝鉴”“副本 × 宝鉴明细”“命中装备列表”三个视图。
- 右侧展示宝鉴概率、期望次数、命中装备、各副本表现和可复制摘要。
- 数据编辑器打开为独立的可调整大小窗口，维护副本、宝鉴和掉落行。
- 默认分栏比例为 20% / 50% / 30%，支持拖拽和本机 UI 偏好持久化。

## 架构

- Tauri 桌面壳负责窗口、文件选择、文件读写、剪贴板和打包。
- React 前端负责页面、状态管理和 shadcn/ui 组件组合。
- 独立 TypeScript domain 层负责数据模型、匹配、权重计算、排序和摘要生成。
- `.zx` 作为 `zx1` 兼容格式；JSON 作为透明、可审查的公开格式。
- 应用默认使用演示数据；真实掉落数据从本机加载并被 `.gitignore` 忽略。
- 数据保存使用临时文件 + 校验成功后替换，避免破坏原文件。

## 计算规则

1. 将属性组合解析为基础属性集合，例如“会专”解析为 `{会心, 专精}`。
2. “命中任一”使用集合交集判断；“同一词条全满足”要求所选属性集合是展开属性集合的子集。
3. 对每个副本 + 宝鉴计算总权重和命中权重。
4. 命中概率 = 命中权重 / 总权重；期望次数 = 1 / 命中概率。
5. 命中组合是去重后的属性组合数量；命中条数是命中掉落行数量。
6. 最佳副本取概率最高者；平均概率为当前候选副本范围内有效概率的算术平均。
7. 推荐排序：最佳概率降序、平均概率降序、命中条数降序、宝鉴名稳定排序。
8. 计算保留精确数值，只在展示层格式化百分比和次数。

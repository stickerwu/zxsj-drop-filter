# 参考界面高保真优化设计

## 目标

以用户提供的第三张主页参考图和掉落表编辑器参考图为唯一视觉基准，在不改变筛选、推荐、数据编辑和 `.zx` 行为的前提下，提高当前 HeroUI v3 界面的信息层级、可读性和桌面工作台质感。

## 视觉方向

- 使用白色和极浅灰作为主要表面，teal 只用于选中态、概率和关键操作。
- 保持紧凑桌面密度，但正文不得小于 12px，主表格正文使用 13px。
- 页面区域不堆叠装饰性卡片；只在指标、装备条目和弹窗行中使用轻量边框与阴影。
- 所有圆角不超过 8px。
- 亮色和暗色主题使用同一结构，不改变尺寸和布局。

## 顶部工具栏

- 高度调整为 72px。
- 品牌图标为 40px，标题 15px，副标题 11px。
- 命令按钮高度 34px，保持轻背景、细边框和 12px 文字。
- 打开数据、导出 JSON、导出 `.zx`、数据编辑、清空条件分别使用紫、灰、蓝、深青、紫色语义图标，但按钮背景仍保持中性。
- 复制标题和主题切换继续使用图标按钮。

## 左侧筛选

- 顶部标题区高度约 68px，右侧增加清空筛选图标按钮。
- 属性和装备部位使用三列矩形选择项，高度 34px。
- 未选中项为浅灰背景和深色文字；选中项为 teal 实心背景和白色文字。
- 选中计数使用小型圆角计数标记。
- “全选 / 不限”使用紧凑文字命令。
- 属性匹配使用清晰的 teal 单选圆环；未选中圆环必须可见。
- 限定副本使用单列整行选择项，高度 34px、文字左对齐、占满可用宽度。
- 计算说明保留在底部，但降低视觉优先级。

## 中央结果

- 条件摘要高度约 56px，第一行显示条件，第二行显示命中数量。
- Tabs 高度 48px，图标和文字使用 12px，选中项为 teal 文字和 2px 下划线。
- 三张表的表头必须使用不透明浅灰背景、12px 半粗字体和清晰前景色。
- 表格正文使用 13px，行高约 46px。
- 表格分隔线使用低对比边框；选中行使用浅 teal 背景。
- 推荐概率使用绿色实心 Chip。
- 推荐表保留八列，不改变列含义、排序或点击选中行为。

## 右侧详情

- 顶部显示“推荐宝鉴”、名称和最佳概率。
- 详情首行使用奖杯/闪光图标和副本信息。
- 命中概率、期望次数两个指标块增加不同语义图标。
- 装备条目改为 52–60px 高的紧凑卡片。
- 图标按装备部位稳定映射，而不是按数组序号循环：
  - 头部、衣服、手部、腰带、腿部、脚部、护符、护佩、法印、令牌、项链、腕饰、天灵、地宝分别使用可辨识的 Lucide 图标和固定色。
- 卡片标题使用 12px 半粗文字，展开属性使用 11px 辅助文字，权重放在右侧小标签中。
- 各副本表现保留紧凑列表，不与装备条目争夺视觉重点。

## 掉落表编辑器

- 保持固定 720px 高度和视口上限，不随内容变化。
- 弹窗宽度约 1020–1100px，圆角 8px。
- 标题区高度约 72px，左侧增加宝石图标；说明文字使用 11px。
- 内容区使用 `220px + 1fr` 两列。
- 左列不再使用厚重内嵌卡片，改为右边框分隔；选择器高度 40px。
- 新增掉落行和全部标为已核对按钮高度 40px，后者使用浅 teal 背景。
- 右侧不显示额外表头条，直接展示间距统一的编辑行。
- 每行高度约 46px，列为部位、属性、权重、展开属性、核对、删除。
- 核对使用绿色方形 Checkbox，删除使用浅红图标按钮。
- 行间距 8px，控件高度 38px。
- 页脚高度约 72px，取消和保存按钮右对齐；保存按钮为 teal 实心，错误信息在左侧。
- 下拉菜单只在当前选中项显示一个勾。

## 测试与验收

- 增加筛选选择态和限定副本整行布局的 DOM 合约测试。
- 增加结果表表头与行密度的 DOM 合约测试。
- 保留编辑器选择和权重校验测试，并继续验证下拉只显示一个勾。
- 运行 `pnpm test`、`pnpm lint`、`pnpm build` 和 `git diff --check`。
- 在 1440×900 和 1180×720 检查亮色、暗色、三个结果 Tab 和编辑器。
- 不接受表头文字低对比、控件重叠、横向溢出遮挡主要数据或弹窗页脚移动。

## 非目标

- 不修改掉落数据和副本名称。
- 不修改推荐算法、匹配算法或 `.zx` 序列化。
- 不新增第四种主题。
- 不重新引入 shadcn/Radix 可见组件。

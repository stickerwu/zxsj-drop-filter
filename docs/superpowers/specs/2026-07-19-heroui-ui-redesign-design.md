# HeroUI v3 UI 重构设计

## 目标

将“诛仙高手秘境掉落软件”的可见界面迁移为 HeroUI v3 风格，参考用户提供的工作台截图，重构主页布局、结果 Tab、数据表格、说明文字、右侧详情和掉落表编辑器。同时新增亮色、暗色、跟随系统三档主题，并提供克制、可关闭的界面动效。

本次不修改 `.zx` 协议、掉落数据、筛选算法、推荐排序、Zustand 状态结构或 Tauri 后端。

## 技术方案

- 使用 HeroUI v3 作为主要可见组件库。
- 将 Tailwind CSS 从 v3 升级到 HeroUI v3 所需的 Tailwind CSS v4。
- 使用 HeroUI 的 Button、Tabs、Table、Modal、Select、Tooltip、Chip、Dropdown、Switch 等组件。
- 保留 `react-resizable-panels`，继续提供左、中、右三栏拖拽调整。
- 保留 Zustand、React、TypeScript、Tauri、sonner 和现有业务模块。
- 不同时维护两套视觉组件。完成迁移后删除不再使用的 shadcn/Radix 包装组件和依赖。

## 主页结构

### 顶部工具栏

- 左侧显示应用图标、名称和一句简短副标题。
- 右侧依次放置打开数据、导出 JSON、导出 `.zx`、数据编辑、清空条件和主题菜单。
- 主题菜单提供“亮色”“暗色”“跟随系统”三个选项，并使用太阳、月亮、显示器图标。
- 工具栏保持固定高度，按钮使用统一尺寸和轻量背景，不因窗口宽度变化产生跳动。

### 左侧筛选区

- 筛选区使用低对比 Surface，而不是层层卡片嵌套。
- 标题、辅助说明、计数和操作按钮形成明确文字层级。
- 属性和装备部位使用紧凑的可切换按钮。
- 激活态采用 teal 主色和轻微填充，不使用高饱和大色块。
- “全选”和“不限”使用轻量文本按钮。
- 属性匹配保留“命中任一”和“同一词条全满足”。
- 计算说明缩小字号，降低对比度，避免抢占主要操作视觉。

### 中央结果区

- 顶部条件摘要使用单行或两行紧凑信息条。
- 三个 Tab 使用图标、文字、激活下划线和柔和背景反馈。
- Tab 内容切换使用短距离淡入位移动效。
- 推荐宝鉴表格保持：排名、宝鉴、最佳概率、最佳副本、平均概率、命中组合、命中条数、预览。
- 副本明细表保持：排名、副本、宝鉴、命中概率、期望次数、命中/总数、预览。
- 命中装备表保持：副本、宝鉴、部位、属性、概率、期望次数、权重、展开属性。
- 表头固定，表格区域独立滚动，行高和列宽稳定。

### 右侧详情区

- 顶部显示推荐宝鉴名称和最佳概率。
- 概率与期望次数采用两个并列摘要块。
- 命中装备使用紧凑列表，每条包含部位、属性、展开属性和权重。
- 不同装备可使用少量语义色图标，但不使用随机颜色或大面积渐变。
- 各副本表现保持扫描友好的单行布局。

## 掉落表编辑器

- 使用 HeroUI Modal。
- 桌面默认高度约 720px，并限制在视口范围内；内容变化不能改变弹窗整体尺寸。
- 标题区、内容区、底部操作区固定分层。
- 左侧为副本、宝鉴选择及新增/批量核对操作。
- 副本、宝鉴、装备部位和属性全部使用 HeroUI Select。
- 属性只能从 10 个合法属性项中选择，选择后自动更新展开属性。
- 右侧装备行区域独立滚动。
- 底部“取消”和“保存并应用”始终可见。

## 主题系统

- 支持 `light`、`dark`、`system` 三种模式。
- 默认使用 `system`。
- 用户选择保存在本地，下次启动继续使用。
- 系统主题变化时，`system` 模式实时响应。
- 主题切换期间仅对颜色、背景、边框和阴影做短时过渡，避免布局动画。
- 暗色主题使用中性深灰与 teal 强调色，不使用大面积深蓝。

## 动效

- Tab 内容：120–160ms 淡入并上移少量距离。
- Modal：淡入和轻微缩放。
- 表格行：hover 背景和操作图标渐显。
- 按钮：按压缩放不超过 0.98。
- 主题：150–200ms 色彩过渡。
- 检测 `prefers-reduced-motion`，用户要求减少动画时关闭非必要动效。

## 状态与数据流

- 主题状态独立于掉落业务状态，不进入 `app-store`。
- 筛选、推荐、编辑和导入导出继续使用现有 Zustand 与 domain 函数。
- HeroUI 组件仅负责展示与交互，不复制业务计算。
- Modal 保存时仍通过 `setDataset` 更新当前数据集。

## 错误处理

- 导入失败继续使用 sonner 显示错误。
- 下拉选项为空时显示明确占位文本，并禁用依赖操作。
- 权重必须为正数；无效输入不能保存。
- 无推荐结果时三个表格显示统一空状态。

## 测试与验收

- 保留现有 14 个领域和数据测试。
- 增加主题模式解析与持久化测试。
- 增加数据编辑器选择副本、宝鉴、部位、属性的交互测试。
- 增加三个 Tab 切换和表格关键列渲染测试。
- 运行 `pnpm test`、`pnpm lint`、`pnpm build`。
- 使用桌面 1440×900 和最小窗口 1180×720 检查无重叠、无溢出和无布局跳动。
- 完成 Tauri NSIS 安装包构建。

## 非目标

- 不重新设计掉落算法。
- 不新增在线更新、账号、云同步或遥测。
- 不修改原始数据格式。
- 不增加更多主题配色，只提供亮色、暗色和跟随系统。

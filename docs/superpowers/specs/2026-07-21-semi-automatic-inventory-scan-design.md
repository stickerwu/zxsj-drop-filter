# 半自动连续库存扫描设计

## 目标

在不向《诛仙世界》发送鼠标或键盘输入的前提下，减少玩家重复按
`Ctrl+Shift+F8` 的操作。玩家只需手动滚动库存；工具检测画面停稳后自动采集、
OCR、重叠配准并合并清单。

## 安全边界

- 继续仅使用 Windows.Graphics.Capture 读取游戏窗口画面。
- 不自动激活窗口，不调用 `SendInput`，不滚动、不点击、不读取游戏内存。
- 全局热键和“采集当前页”按钮保留，作为首次采集和识别失败后的补采入口。
- 原始画面与探测特征只保存在当前扫描会话内存中。

## 后端状态机

- 新增轻量探测命令，每约 250ms 捕获一帧并从库存所在的右侧区域生成低分辨率
  灰度特征。
- 相邻特征差异超过阈值时状态为 `scrolling`，并重新开始停稳计时。
- 画面连续稳定至少 350ms，且与最后一次尝试采集的画面不同，返回
  `shouldCapture: true`。
- 首次进入库存页同样需要稳定 350ms 后自动采集。
- 每个静止画面只自动尝试一次。OCR、锚点或重叠配准失败后不循环重试，玩家
  滚动画面或使用手动采集即可继续。
- 捕获尺寸变化、最小化、黑帧和会话过期继续使用现有错误处理。

## 前端交互

- 扫描会话开始后默认开启“连续扫描”。
- 左栏显示 `等待画面稳定 / 检测到滚动 / 正在识别 / 已采集当前画面 /
  识别失败，请调整后重试` 状态。
- 引导文案改为“手动滚动，停稳后自动采集”，并说明每次保留至少一行重叠。
- `Ctrl+Shift+F8` 与“采集当前页”继续可用。
- 自动探测只在核对窗口打开、扫描会话有效、连续扫描开启且当前没有 OCR 任务时运行。

## 测试与发布

- Rust 单元测试覆盖首次停稳、滚动后停稳、同画面不重复触发和采集失败后的抑制。
- 前端测试覆盖默认开启连续扫描、自动触发采集、关闭后停止探测和新状态文案。
- 回归现有库存核对、求解器、掉落筛选、主题和更新功能。
- 版本更新为 `0.6.0`，更新 README 与 CHANGELOG，构建签名 NSIS 安装包并推送
  GitHub/Gitee 的 `v0.6.0` Release。

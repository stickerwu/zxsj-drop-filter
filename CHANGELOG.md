# Changelog

## 0.5.4 - 2026-07-21

- 命中装备列表的宝鉴、部位表头增加 HeroUI 多选筛选。
- 宝鉴与部位筛选按 AND 逻辑组合，仅影响命中装备列表，不改动其他结果页签或全局筛选条件。
- 活跃筛选显示选中数量，支持一键清除，并为无匹配结果提供独立空状态提示。

## 0.5.3 - 2026-07-21

- 将库存扫描核对窗口的“手动新增”按钮固定到机巧石与匠心石页签栏最右侧。
- 压缩弹窗标题区、页签工具栏和底部操作区高度，重新整理图标、标题与说明的横向对齐。
- 移除标题区、页签栏和底部操作区之间多余的横向分割线，改用背景层级和主体阴影保持区域区分。

## 0.5.2 - 2026-07-20

- 修复游戏保护拒绝 `PROCESS_VM_READ` 后 dev 与正式版均无法列出《诛仙世界》窗口的问题，改用 Windows 有限查询权限读取可执行文件路径，不读取游戏内存。
- 游戏进程名匹配兼容带或不带 `.exe` 的返回值，同时继续校验“诛仙世界”窗口标题。
- 库存扫描窗口增加自动重试与手动刷新，并明确提示“请打开个人游戏角色的天工机巧盘页面”。
- 核对窗口完整迁移到 HeroUI `Select`、`NumberField`、`Switch`、`Tabs` 和 `Input`，增强亮暗主题对比度、三栏层级、空状态与低置信度提示。
- 修复扫描会话启动时后端时间戳不符合前端 ISO datetime 校验而报错的问题。
- 在真实运行的游戏窗口上验证窗口枚举、Windows.Graphics.Capture 非黑帧捕获和扫描会话启动。

## 0.5.1 - 2026-07-20

- 修复大体积 OCR 安装包上传 Gitee 时的发布流水线失败。
- Gitee Release 附件上传对网络错误、限流和服务端错误自动重试三次。
- Gitee 大附件持续失败时仍发布 Gitee Release 与更新清单，并将安装包地址回退到同版本 GitHub Release。

## 0.5.0 - 2026-07-20

- 天工机巧盘工具栏新增“库存扫描”，通过 Windows.Graphics.Capture 捕获未最小化的《诛仙世界》窗口。
- 扫描会话期间注册 `Ctrl+Shift+F8`，玩家手动切换页签和滚动列表，软件不会自动点击、滚动或向游戏发送输入。
- 随安装包提供 PP-OCRv5 中文 ONNX 模型，全程离线识别库存总数、五类形状、五行、品质、主属性、灵蕴和标记。
- 新增固定高度三栏库存核对窗口，支持多窗口选择、逐项修正、手动增删、低置信度提示和完整性阻断。
- 相邻页面通过垂直图像配准合并，保留外观和属性完全相同但位置不同的多块石头。
- 核对完成后一次性同步五类普通机巧石数量并清空旧求解结果，完整石头清单保存在本机应用数据目录。
- 原始截图和卡片裁剪仅驻留当前进程内存，不写磁盘、不上传；游戏窗口最小化、黑帧、锚点失效或分辨率变化会终止本次采集。
- 机巧石库存上限从 99 扩大到 999，配置文件版本仍为 1。

## 0.4.0 - 2026-07-20

- 应用更名为“诛仙高手工具箱”，顶部增加“秘境掉落 / 天工机巧盘”双模式切换并记住上次模式。
- 新增 7×6 异形天工机巧盘，支持点击、键盘和按住拖拽编辑任意解锁区域。
- 新增匠心石必放规则，以及正方形、L 型、T 型、一字型、J 型库存配置。
- 使用独立 Web Worker 和 SAT 约束求解器本地计算，支持旋转不镜像、库存上限、方案去重和最多 5000 个方案。
- 新增单盘上一解/下一解浏览、每解用石统计、无解与配置错误提示。
- 新增版本化机巧盘 JSON 配置、自动本地保存、导入、导出和恢复默认。
- 保留现有秘境掉落筛选、掉落表编辑器、主题切换和 Gitee 优先自动更新行为。
- Windows 应用、安装器、卸载器和快捷方式继续使用现有程序图标。

## 0.3.1 - 2026-07-20

- 去除选中项的大面积背景，仅在右侧显示当前主题对勾，并保留轻量悬停和键盘焦点反馈。
- 修复 HeroUI 指示器默认定位导致对勾与左侧主题图标重叠的问题。
- 补充主题菜单无障碍名称、主题切换和唯一选中标记回归测试。

## 0.3.0 - 2026-07-20

- 增加 GitHub Actions 自动构建 Windows NSIS 安装包。
- 自动创建 GitHub Release 和 Gitee Release，并上传一致的 NSIS 安装/更新包、签名和 SHA-256 校验文件。
- 接入 Tauri 签名 updater，默认使用国内 Gitee 更新源，Gitee 不可用时回退 GitHub。
- 顶部工具栏增加检查、下载进度、更新就绪、安装失败和手动重试状态。
- 更新下载完成后提供“稍后安装”和“立即安装并重启”，掉落编辑器打开时延后显示确认框。
- `v0.2.1` 用户需要手动安装一次 `v0.3.0`，后续版本才能使用应用内自动更新。
